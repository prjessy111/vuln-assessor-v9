# 산출물 9 — Mandate / Harness
## AI 진단 에이전트의 역할 · 목적 · 수행 범위

> 3-세트 문서 ①/③ — [Mandate] · [Context](산출물_10_Context.md) · [Directive](산출물_11_Directive.md)

---

## 1. 정체성 (Role)
**SecuMS 점검 결과를 raw evidence 기반으로 독립 재판정하는 AI 검증 엔진.**
- SecuMS의 자체 판정값·취약 메시지는 입력에서 제외하고, **수집된 raw 데이터만** 근거로 판정한다.
- 사람의 수작업 정합성 대조를 **AI 자동 검증**으로 대체한다.

## 2. 목적 (Purpose)
- **정합성 자동 검증**: SecuMS 결과가 맞는지 항목 단위로 자동 확인.
- **판정불가(N/A) 최소화**: 수집 보강 + 판정 규칙으로 자동 판정률 극대화.
- **정합성 유지**: 정답지(SecuMS 자체판정) 대비 **84~88%(사내 LLM 기준, 실측)** 안정 유지 (단일 고정치 아닌 관리 지표).

## 3. 수행 범위 (Scope)
| 구분 | 내용 |
|---|---|
| 대상 OS | Windows / Linux (금융보안원 전자금융기반시설 **2026 OS 점검**, 약 64항목) |
| 입력 소스 | ① SecuMS raw DB  ② 자체 수집 스크립트(XML) — 두 소스 독립 판정 |
| 수집 방식 | **에이전트리스** — SSH(Linux) / WinRM(Windows). 대상에 상주 데몬 없음 |
| 판정 단위 | 항목별 **취약 / 양호 / 정보제공 / 판정불가** + 사유·증거·조치 |
| 판정 흐름 | **2-pass** — 1차 AI(빠른 분류) → 2차 LLM(정밀 재검토) |
| CVE | rpm 패키지(Linux)/핫픽스(Windows) 기반 **결정론적** 매칭 (KEV 포함) |
| 정합성 | ①SecuMS-AI ②Script-AI ③SecuMS 자체판정(정답지) 3-way 정합성 비교 (정답지 N개 중 M개 = X%) |
| 자율 진단 | 신규 항목 **자연어 편입 → 스크립트 자동 생성 → 실행(SSH/WinRM) → 판정 → Hermes 보고** (Claude Skill 에이전트, 사람 게이트 2개) — [산출물_12](산출물_12_자율진단에이전트.md) |

## 4. 범위 밖 (Out of Scope — 향후 과제)
- 자동 조치(remediation) **실행** — 미지원. 현재는 조치 관리/기록 + (판정 시 **조치 가이드 텍스트** 제시)까지.
- 타 OS(AIX·HP-UX·Solaris) 확장.

> ※ "신규 점검 항목 자동 생성 AI Agent(Claude Skill)"는 당초 범위 밖이었으나 **본 과제 내 구현·실측 완료** → [산출물_12](산출물_12_자율진단에이전트.md). (자연어 항목→스크립트 생성→실행(SSH/WinRM)→판정→Hermes 보고, 사람 게이트 2개)

## 5. 운영 제약 (Harness)
- **판정 LLM**: 사내 LLM(qwen, 무료) — 사내 운영.
- **하드코딩 금지**: 폴백 DB·서버값·시간 등 임의값 금지, 항상 실데이터에서 해석.
- **판정 기준 외부화**: 기준은 코드가 아닌 기준표·프롬프트(`data/report1-2026-checkspec.json`, `srv-secums-crosswalk.json`)에서 주입 → 기준 변경·재현 용이.
