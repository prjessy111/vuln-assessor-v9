# 산출물 11 — Directive
## 결과물의 형식 · 작성 방식 · 출력 기준

> 3-세트 문서 ③/③ — [Mandate](산출물_9_Mandate.md) · [Context](산출물_10_Context.md) · [Directive]

## 1. 출력 형식 (Format)
항목 1건당 **JSON만** 반환 (다른 텍스트 없이):
```json
{
  "verdict":   "취약 | 양호 | 정보제공 | 판정불가",
  "safe_type": "부재양호 | 값준수양호 | \"\"",
  "category":  "보안 카테고리",
  "title":     "한글 제목(15자 이내)",
  "reason":    "판정 사유(구체적으로)",
  "evidence":  "raw 데이터 핵심 증거 1-2줄 인용",
  "recommend": "조치 권고(취약 시 명령/설정). 양호면 빈 문자열",
  "severity":  "상 | 중 | 하"
}
```

## 2. 작성 방식 (Style)
- **reason**: 왜 그 판정인지 구체적 근거. 추상적 단정 금지.
- **evidence**: raw 출력에서 **실제 1–2줄 인용** (가공·창작 금지).
- **title**: 15자 이내 한글.
- **recommend**: 취약 시 적용할 명령/설정. 양호면 빈 문자열.
- **근거 원칙**: 오직 raw evidence에 근거. 기준표(2026)를 raw에 비춰 **결정적으로** 판정.
- **데이터 소스 표시**: 결과에 SecuMS / Script 출처를 명시.

## 3. 출력 기준 (Output Rules)
- **수집실패·권한오류·잘린 출력·미수집 → `판정불가`.** 절대 `양호`로 처리하지 않는다.
- **기준이 명확하면** `정보제공`/`판정불가` 대신 **`취약`/`양호`로 결정**한다.
- **정보제공**: 단순 현황조회로 기준이 불명확할 때만.
- **severity**: 2026 기준의 위험도에 따른다 (상/중/하).

## 4. 2-pass 출력 정책
- **1차 AI**: 명시적 raw 신호일 때만 취약/양호, 모호하면 정보/판정불가 (속도 우선).
- **2차 LLM**: 명령 의도·부재양호 vs 값준수양호·수집 오류까지 검증 후 확정. 근거 부족이면 판정불가.

## 5. 리포트·화면 출력 기준
- **진단 리포트**: 항목별 취약/양호 판정 + 사유·증거·조치, 취약점만 보기, XLSX 다운로드.
- **3-way 정합성**: ①SecuMS-AI ②Script-AI ③SecuMS 자체판정(정답지) 나란히 + **정합성(정답지 N개 중 M개 = X%)** 상시 표시. 불일치만 보기 지원.
- **소요시간·수치 표기**: 사람이 읽기 쉽게(분·초), 모수(전체)+사용/차지 절대값 함께.
- **데이터 정직성**: 샘플/가비지 금지, 실데이터만. 관리 대상(servers.csv) 기준으로만 집계.
