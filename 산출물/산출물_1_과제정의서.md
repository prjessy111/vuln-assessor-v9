# 과제 정의서 — AI 기반 보안 취약점 진단 정합성 자동 검증

## 1. 과제 개요
| 항목 | 내용 |
|---|---|
| 과제명 | AI 기반 보안 취약점 진단 — 정합성 자동 검증 |
| 부서 | 제품전략팀 |
| 대상 시스템 | Vuln Assessor (취약점 진단 시스템) v9 |
| 점검 표준 | 금융보안원 전자금융기반시설 2026 OS 점검 기준 (SecuMS 점검 64항목) |

## 2. 배경 / Pain Point
고객사가 SecuMS(보안 점검 제품) 결과의 **정합성 확인**을 요청할 때, 현재는:
- **정합성 확인이 전부 수작업** — 담당자가 항목별로 일일이 대조, 시간 과다.
- **정합성 판단 기준의 부재** — 판정이 맞는지 검증할 표준·방법이 없어 매번 즉흥적.
- **신규 항목 대응 지연** — 새 점검 항목 요청 시 그 정합성 확인에도 큰 시간 소요.

## 3. 목표
세 가지 점검 결과가 **거의 일치**함을 자동 검증하고, 그 방법을 표준화한다.
1. **① SecuMS raw data 기반 AI 진단** — SecuMS 원본 데이터를 AI/LLM이 독립 판정
2. **② Script 기반 AI 진단** — 자체 수집 스크립트(ps1/sh) 결과를 AI/LLM이 판정
3. **③ SecuMS 자체 판정** — SecuMS 제품의 점검 결과 (정답지)

→ **SecuMS 정답지 대비 정합성 84~88%(사내 LLM 기준, 실측)** 를 검증. 검증되면 SecuMS 미커버 항목까지 AI로 확장.

**+ 신규 핵심 차별 — 자율 진단 에이전트(Claude Skill):** 위 검증 체계 위에, 신규 점검 항목을 **자연어 설명만으로** 정의하면 점검 스크립트 생성 → 실행(SSH/WinRM) → 판정 → Hermes 메신저 보고까지 자동 수행. 사람은 두 게이트(실행 승인·판정 확정)만 검수. (상세 — [산출물_12](산출물_12_자율진단에이전트.md))

## 4. 적용 방식 (Prompt / Context / Harness)
| 구분 | 내용 |
|---|---|
| **Prompt** | 독립 판정 엔진 역할 + 2026 공식 기준(확인방법·기준·조치법) 항목별 주입 + 엄격 판정 원칙 + 과탐 방지 규칙 |
| **Context** | 서버 OS · 수집 raw 증거 · 2026 금융보안원 OS 점검 기준 · SecuMS 정답지(상세리포트) |
| **Harness** | AI 1차(빠른 휴리스틱) + LLM 2차(정밀 재검토) 2-pass · 자동 수집(WinRM/SSH) · 정합성 자동 측정(3-way) · CVE 진단 병행 |

핵심 설계: **수집(raw)과 판정(AI)을 분리**하고, 판정 기준을 코드가 아닌 **프롬프트·기준표(JSON)로 외부화** → 변경·고도화·재현 용이.

## 5. 범위
- 대상 OS: Windows / Linux (CentOS·RHEL 계열)
- 점검 항목: 금융보안원 2026 OS 점검 (SecuMS 점검 64항목) + CVE(rpm/핫픽스 기반)
- LLM: 사내 LLM(qwen) — 무료·사내 운영
