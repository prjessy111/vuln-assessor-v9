## 2. 도메인 지식 (Domain)
- **OS 보안 점검 영역**: 계정 관리 · 서비스/데몬 · 파일 권한 · 패치 관리 · 로그/감사 · 네트워크 접근제어.
- **CVE/KEV**: rpm 인벤토리·핫픽스 기반 실제 취약 매칭 (EternalBlue·Zerologon·PrintNightmare·sudo Baron Samedit 등).
- **데이터 소스 성격**: SecuMS raw(DB 명령/SQL 결과) / Script XML(배포 스크립트 실행 결과) — 둘 다 판정값 없음, raw 출력만 근거.

## 3. 업무 기준 (Doctrine — 판정 원칙)

### 3.1 엄격 판정 6원칙
1. 2026 기준 최우선. 임의 "표준/필수/정상" 단정 금지.
2. 보안 강화값이 기준 미달·미설정/기본값이면 취약.
3. 특권 그룹(Administrators/wheel)에 기본 외 사용자 있으면 취약.
4. 비밀번호 장기 미변경·만료 미설정(99999)이면 취약.
5. 권한/SUID/SGID/umask 기준 초과면 취약 (표준 유틸 임의 화이트리스트 금지).
6. 수집된 모든 값 검토, 하나라도 위반이면 취약. 기준 명확하면 취약/양호로 결정.

### 3.2 과탐 방지 규칙 (정합성 핵심)
- **(가) 더 엄격한 권한은 양호** — shadow 0000/0400, 로그 0600은 강함 → 양호.
- **(나) 잠긴 계정은 양호** — shadow 2번째 필드 `*`,`!`,`!!`,`*LK*`. 빈 암호는 `::` 일 때만 취약.
- **(다) OS 기본 시스템 계정(UID/GID<1000)** 은 양호. 엄격 판정은 사람 계정(UID≥1000)에만.
- **(라) loopback/내부대역 전용 LISTEN은 외부노출 아님 = 양호** — 배너/하드닝 설정 미확인이어도 내부 전용이면 양호.
- **(마) raw 미수집(빈 컬럼·Rows=0)은 취약 단정 말고 판정불가** — 단 대상 실제 부재면 기준대로.
- **(바) 항목 범위 준수** — 빈암호 항목에서 만료기간으로 취약판정 금지.
- **(사) 미설치 → 부재양호** — 서비스 바이너리/설정/포트가 raw에 전혀 없으면 미설치=부재양호. 명령 미인식("인식되지 않습니다"/"not recognized")=도구 미설치=부재양호.
- **(아) (사)의 예외** — ① 실행 중인데 설정만 못 읽으면 수집오류=판정불가. ② 대체 데몬(sendmail↔postfix, apache↔nginx) 살아있으면 부재양호 금지. ③ 빈암호·홈디렉 등 항상 존재 대상의 빈 출력은 수집실패=판정불가.

### 3.3 핵심 구분
- **부재양호**: 점검 "객체 자체"(파일/서비스/패키지)가 없을 때만.
- **판정불가**: 점검 미실행·권한오류·잘린 출력·raw 부족 — **양호로 보지 말 것**.
- **미설정 ≠ 부재양호**: 설정값/레지스트리 키 미설정은 기본값 동작 → 기본값 안전성으로 판정.
