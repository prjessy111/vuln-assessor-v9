## 2. AI 활용 충실도 (Prompt / Context / Harness)
- **Prompt**: 2026 금융보안원 공식 기준(확인방법·기준·조치법) 항목별 주입 + 엄격 판정 6원칙 + 과탐 방지 6규칙. 판정 기준을 코드가 아닌 프롬프트·기준표로 외부화.
- **Context**: 서버 OS·raw 증거·2026 기준·SecuMS 정답지 결합.
- **Harness**: AI 1차(빠른 분류) + LLM 2차(정밀 재검토) 2-pass, 자동 수집(WinRM/SSH), 3-way 정합성 자동 측정, CVE 병행, 사내 LLM(qwen) 운영.

## 3. 재현성 / 검증
- 정답지(SecuMS 상세리포트) 대비 항목별 일치율 정량 측정 (3-way 화면).
- 표준 준거: 금융보안원 전자금융기반시설 2026 OS 점검.
- 수집/판정 분리 — 증거 없으면 억지 판정 대신 "판정불가"(과탐 방지).
- 동일 프롬프트·기준표(JSON)로 재현 (사내 LLM 기준).

## 4. 확장성
- **신규 항목**: 스크립트 배포·AI 진단으로 즉시 점검 (SecuMS 미커버 항목 확장).
- **타 OS / CVE**: AIX/HP-UX/Solaris, MSRC/NVD 피드 동기화로 확장 가능.
- **비용 효율**: 사내 LLM(qwen, 무료) 운영 — 추가 비용 없이 자동 검증.

## 5. 핵심 인사이트 (의사결정 근거)
- **mock(룰)↔LLM 대비** = "AI 없이는 자동 진단 불가(N/A 61%)"의 정량 증거.
- 정합성 한계의 진짜 원인 = 모델이 아니라 **수집 완전성(full) + 판정기준 정의 차이** — 데이터·기준으로 해결.
