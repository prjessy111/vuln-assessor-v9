AI 기반 보안 취약점 진단 — 정합성 자동 검증

고객사 보안점검(SecuMS) 결과의 정합성 확인을 사람 수작업에서 AI 자동 검증으로 전환
제품전략팀대상 시스템 · Vuln Assessor v9 점검 표준 · 금융보안원 전자금융기반시설 2026 OS 점검 대상 OS · Windows / Linux
고객사가 요청하는 SecuMS 점검 결과의 정합성 확인을, 담당자가 항목별로 일일이 대조하던 수작업에서 AI/LLM 자동 검증으로 전환했습니다. 결과적으로 SecuMS 정답지 대비 정합성 84~88%(사내 LLM 기준)를 달성하고, 자동 판정 불가(N/A) 항목을 대부분 제거하여 사람 개입 없는 자동 검증 체계를 확보했습니다.

2. 배경 — 왜 필요했나 (비즈니스 문제)

  • 고객 정합성 확인 요청이 전부 수작업 — 담당자가 70여 개 항목을 일일이 대조, 서버마다 반복 → 인력·시간 과다.
  • 정합성 판단 기준의 부재 — 판정이 맞는지 검증할 표준·방법이 없어 매번 즉흥적, 품질 편차 발생.
  • 신규 점검 항목 대응 지연 — 새 항목 요청 시 정합성 확인에도 큰 시간 소요.

3. 무엇을 했나 (솔루션 요약)

세 가지 점검 결과를 항목 단위로 비교해 정합성을 자동 검증하는 체계를 구축했습니다.

핵심 설계: 수집과 판정을 분리하고, 판정 기준을 코드가 아닌 금융보안원 공식 기준표·프롬프트로 외부화 → 기준 변경·고도화·재현이 용이합니다.

5. 비즈니스 가치

  • 고객 대응 속도·품질 — 정합성 확인 요청을 즉시·일관된 기준으로 처리.
  • 인력 효율 — 반복 수작업 제거, 담당자는 예외·고난도 항목에 집중.
  • 비용 효율 — 사내 LLM(무료)과 결정론적 CVE 진단으로 추가 비용 없이 운영.
  • 에이전트리스(Agentless) — 대상 서버에 에이전트 설치 없이 SSH/WinRM로 수집·검증. 도입 부담·운영 리스크 최소화 (핵심 강점).
  • 확장 기반 — 검증된 체계를 SecuMS 미커버 신규 항목·타 OS로 확장 가능.

6. 정합성에 대한 이해 (경영진 참고)

AI 기반 자동 판정은 단일 고정 수치가 아닌 "관리 지표"로 보는 것이 정확합니다. 이유:
  • 측정 층위가 여러 개 — 판정불가율 / AI↔정답지(2-way) / 세 결과 모두일치(3-way). 엄격한 지표일수록 수치가 낮게 보입니다.
  • AI 고유의 미세 변동 — 경계 항목은 실행마다 ±수% 흔들립니다(룰과 달리 100% 동일 재현은 불가).
  • 의도된 엄격 판정 — 일부 항목은 안전을 위해 SecuMS보다 엄격히 봅니다(놓침 방지). 무조건 일치시키면 오히려 위험.
→ 운영 방침: 100% 고정이 아니라 판정불가 최소화 + 정답지 대비 정합성 84~88%(사내 LLM) 안정 유지를 목표로, 리포트 화면에 정합성(모수 N개 중 M개)을 상시 표시해 수치로 관리합니다.
현재 측정 기준선(실측): ① SecuMS raw AI 84.5% (정답지 58개 중 49개 일치) · ② 자체수집 AI 88.3% (60개 중 53개 일치). — 실행별 ±수% 변동(LLM 특성). 리포트 화면 수치 기준.
