## 1. CVE — 설치 자산(SW) + OS 패치, 멀티 OS, 자동갱신 (핵심 작업)

### 철학 (사용자 강조)
- **CVE는 우리가 주인(권위).** 외부 보고서(docx 등)는 **참고·교차검증용이며 틀릴 수 있음** — 답은 우리가 산출.
- DB를 docx에 맞추지 말 것. **NVD 출처**로 넓게. 우리 데이터가 docx보다 많아야 함.
- 강점 = **CVE(설치자산) + 3-way 정합성 + 자율에이전트 자동화** = SecuMS가 못 하는 것.

### 구현된 것
| 영역 | 파일 | 내용 |
|---|---|---|
| Windows 설치 SW 파서 | `src/cve/winScanner.js` | `extractWindowsSoftwareFromScriptXml` — script XML의 **INV-SOFTWARE**(레지스트리 Uninstall) 파싱 → `{name,version,publisher}`. jessy107 68건 검증 |
| Windows SW 매칭 | `src/cve/winScanner.js` | `matchWindowsSoftware` — name+version, 문자접미사·버전범위, `cve-software-curated.json` 사용 |
| Linux Ubuntu/Debian | `src/cve/scanner.js` | `extractPackagesFromDpkgText` + `sliceDpkgSection` — **dpkg -l** 파싱. rpm 없으면 dpkg fallback. OS 자동판별 |
| Ubuntu 패키지명 정규화 | `src/cve/scanner.js` | `DEB_NAME_ALIAS` — libssl1.1→openssl, zlib1g→zlib, libexpat1→expat, libpcre3→pcre 등 |
| 문자접미사 버전 | `src/cve/matcher.js` | `compareVersions`에 끝자리 알파 타이브레이커 (OpenSSL 1.0.2o vs 1.0.2p). rpm 'pN'엔 영향 없음 |
| 공통 SW CVE DB | `data/cve/cve-software-curated.json` | **실제 NVD CVE 27건** (Log4j/OpenSSL/zlib/expat/SQLite/PCRE/Postgres/httpd/tomcat/curl/struts/spring/libwebp/git/bash/7zip/putty/tinyxml). Win·Linux 공유 |
| Linux 매처 병합 | `src/cve/matcher.js` | `loadCveDb`가 centos-curated + software-curated 합침 (include_from:true 주입) |
| 자동갱신 | `server-mock.js` | `startCveAutoSync()` — 부팅 시(24h초과)+매일 `sync-cve.js` spawn. `CVE_AUTO_SYNC=0`로 끔. 갱신 후 캐시 무효화 |
| 화면 | `src/views/cve/scan_windows.ejs` | 설치 SW 인벤토리 펼침표 + 소프트웨어 CVE 매칭표 |

### CVE 라우팅 버그 수정 (중요)
- `server-mock.js` `_sniffScriptOs` / `resolveCveSource`: Windows(`<os>Microsoft Windows...`/`<os_family>windows`)를 인식 못 해 **리눅스 rpm 스캐너로 오라우팅**되던 버그 수정. unknown 시 서버 os_type 폴백.

### 데이터 소스 적합성 (확인됨)
| | OS패치 CVE | 설치SW CVE |
|---|---|---|
| SecuMS Linux DB (rpm) | ✅ | ✅ |
| SecuMS **Windows** DB | ✅ 핫픽스 | ❌ **SW 인벤토리 테이블 없음** → script(INV-SOFTWARE) 필요 |
| script XML | ✅ | ✅ (Win INV-SOFTWARE / Linux rpm·dpkg, -Full 필요) |

### 자동갱신 출처/방법
- NVD(`nvd.nist.gov`) modified/recent/full + CISA KEV(`cisa.gov`) + EPSS. `scripts/sync-cve.js`.
- 앱 내장 일일 스케줄(위) 또는 수동: 웹 `/cve/sync` 버튼, CLI `node scripts/sync-cve.js [--full|--from-file <p>]`.
- **인터넷 필요.** 폐쇄망은 `--from-file`. 최신성 = 마지막 성공 동기화 시점.

### 정직한 잔여 한계
- SW CVE DB 27건은 **베이스라인** — NVD 자동갱신으로 계속 확대해야 docx 능가.
- 동기화된 `cve-db.json`(1604건)은 affected(CPE)가 비어 매칭엔 미사용 → **매칭은 큐레이션 DB(affected_versions) 기준**. NVD CPE→affected_versions 변환은 향후 과제.
- 우분투 백포트 release(1ubuntu2.x)는 backport 패치판정 미적용 → 과탐 가능(AI/LLM 층에서 정제).

## 2. WinRM (Windows 원격 진단) — 사전작업.md 참조

- **핵심**: 대상마다 ① 컨트롤러 TrustedHosts 등록 ② 대상에 **Basic 인증 + AllowUnencrypted** 켜기 (앱은 nodejs-winrm=Basic 사용. Invoke-Command 테스트 성공 ≠ 앱 동작).
- 완료: jessy107(.107), jessy147(.147 = dolf-w2022-147) Basic ON. jessy167(.167) SSWindows DB 있음.
- 한 줄 명령(내 PC 관리자 PS):
  ```powershell
  $cred = Get-Credential; Invoke-Command -ComputerName <IP> -Credential $cred -ScriptBlock { winrm set winrm/config/service/auth '@{Basic="true"}'; winrm set winrm/config/service '@{AllowUnencrypted="true"}' }
  ```
- 147은 **servers.csv 미등록** — 비번/자산번호 확보 시 추가 필요.

### 신규/수정 분석 모듈 (src/cve/)
- `winScanner.js`: extractJars/NativeLibs + parseJarFilename(빌드접미사 보정) + matchWindowsSoftware(basis 근거·backport_risk·verdict)
- `scanner.js`(Linux): dpkg + rpm -qa(NVRA) + jar 파서 + distroAdvisory 연동(env)
- `nvdCpe.js`: **NVD cve-db.json(CPE)→매처 형식 어댑터** (확장 핵심)
- `sbomParser.js`: CycloneDX/SPDX → {name,version} (purl)
- `kevCrosscheck.js`: 설치SW vs CISA KEV 제품 (GENERIC 스킵리스트로 오탐 억제)
- `judge.js`: judgeSoftwareAll — **백포트 인지 판정**(번들=LIKELY, 설치프로그램=NEEDS_REVIEW)
- `distroAdvisory.js`: RHEL/Ubuntu 보안 advisory로 **백포트 결정론 판정**(과탐 제거). `CVE_BACKPORT_CHECK=1`로 활성
- `cve-software-curated.json`: 실제 NVD CVE 33건 (OpenSSL/zlib/expat/sqlite/pcre/log4j/commons*/pdfbox/pgjdbc/httpd/tomcat/struts/spring 등)
