Windows CVE 자동 진단 동작 방식

  • Stage 1 — 인벤토리 추출: OS 빌드 + 설치 핫픽스(2개 KB) + 서비스 상태
  • Stage 2 — 패치 판정: 빌드 UBR ≥ 수정 UBR 이거나 수정 KB가 설치됨 → 패치, 아니면 미설치
  • Stage 3 — 조건 평가: 일부 CVE는 특정 서비스(SMB·Spooler·Netlogon 등) 실행 여부로 공격 표면 가중
  • Stage 4 — 우선순위: CISA KEV → 미패치 → CVSS 순

※ 본 기능은 큐레이션된 핵심 Windows Server CVE만 포함합니다. 운영 환경에서는 Microsoft 보안 업데이트 가이드(MSRC) / NVD 피드를 동기화하여 KB-CVE 매핑을 최신으로 유지해야 합니다. 설치 프로그램(서드파티 SW) 단위 CVE는 인벤토리 수집 확장이 필요합니다.
