리포트3 — 타사 점검정책, 우리 스크립트로 커버 가능한가?

대상: 리포트3.xlsx(타사 OS 보안 점검 카탈로그) vs vuln-assessor FSI/금보원 스크립트 · 2026-06-30

1. 결론 — 가능 (우리가 superset)

타사 리포트3 점검항목은 우리 스크립트의 부분집합이다. 항목 수만 봐도 우리가 2~3배 많고, 타사 항목(UID 0·패스워드·파일권한·umask·SNMP·방화벽·기본공유·백신·로그·sshd·배너 등)은 전부 우리 FSI/금보원 baseline에 존재한다. → 스크립트 재작성 없이, 우리 항목을 선택·매핑해 타사 정책으로 산출 가능. 빠진 소수 항목만 자율에이전트로 추가.

4. 타사 정책 적용 방법 (재작성 불필요)

  1. 매핑 테이블 작성: 타사 항목ID ↔ 우리 SRV/수집키 (대부분 1:1)
  2. 우리 스크립트로 전체 수집(이미 superset) → 타사 항목만 선택 출력
  3. 타사 고유 점수/등급(1~5점, A 95%+ 등) 룰을 리포트 룰셋으로 추가
  4. 우리에 없는 소수 항목만 자율에이전트로 description 주고 스크립트 생성

5. 정직한 한계

  • 본 검토는 항목 수·대표 항목 대조 기준 — 47/42개 전 항목 1:1 매핑은 항목별 정밀 대조 작업 필요(제목/기준 표현 차이).
  • 타사 고유 항목(있다면)·점수가중치·등급기준은 리포트 룰셋으로 별도 반영해야 함.
  • OS 버전 커버(Solaris 9~11, AIX 5.1~7.1, HP-UX 11.x)는 대상별 실측 검증 권장.
