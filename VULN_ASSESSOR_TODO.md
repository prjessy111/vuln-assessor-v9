# Vuln Assessor — 향후 작업 지시서 (Claude Code용)

> 이 문서는 Claude Code가 읽고 작업에 착수하기 위한 TODO/로드맵이다.
> 프로젝트: `prjessy111/vuln-assessor-v9` (현재 v9.21)
> 스택: Node.js / Express / EJS / MySQL (Windows 10 dev, Windows Server 2012R2 DB)
> LLM 백엔드: 사내 LSAP (`lsap.lsware.net`, `qwen3.5-122b-fast`) — 외부 전송/비용 회피 목적
> 핵심 정체성: **"진단 도구가 아니라, 진단 결과를 전수 자동 검증하는 시스템"**

---

## 0. 작업 원칙 (모든 작업에 공통 적용)

- **독립 판정 유지**: mock/LSAP는 SecuMS 판정을 정답으로 쓰지 않고 raw 데이터로 독립 판정한다. (검증 시스템의 존재 이유)
- **패치 전달**: 변경 파일만 zip으로. 전체 프로젝트 zip 금지. `npm install` 최소화.
- **코드 구조 보존**: 기존 구조를 불필요하게 갈아엎지 말 것. 최소 변경으로 기능 추가.
- **용어 통일**: 행위=「진단」(전체 진단/예약 진단), 대상·현황·이력=「점검」 허용, 실행 시점=「예약/즉시」(자동/수동 금지).
- **데이터 외부 전송 주의**: raw 점검 데이터는 기본 LSAP(내부)로 처리. 외부 API(Claude 등)는 제한적·선택적으로만.
- **사람 확인 게이트**: 자동 생성 스크립트 실행 전, 자동 판정 불일치 항목에는 사람 승인 단계를 둔다.

## 4. 정확도·검증 신뢰성 (수치로 증명)

### 4-1. 검증 효과 정량화
- [ ] "독립 검증으로 오탐 N건 제거" 산출·표시
- [ ] "3-way 일치율 X%, 불일치 Y건 재검토" 대시보드화
- [ ] mock 일치율을 "독립 판정 작동 근거"로 리포트에 명시

### 4-2. 오탐/미탐 지표 자동 산출
- [ ] 재현율(Recall) = 정탐수 / 정답지
- [ ] 정밀도(Precision) = 정탐수 / 탐지수
- [ ] 오탐율(False Discovery Rate) = 1 - 정밀도
- [ ] 위 지표를 대시보드에 노출

### 4-3. LSAP vs Claude 벤치마크 (도입 판단용)
- [ ] 대표 항목 수십 개에 정답지(ground truth) 구축
- [ ] 동일 기준으로 LSAP vs Claude 정밀도·재현율·일치율 비교
- [ ] "Claude로 정확도 향상" 가설을 실측으로 검증 (체감 90% 여부 확인)

### 4-4. 하이브리드 검증 구조  (§3 교차검증으로 일부 구현)
- [x] 전수 진단 = LSAP (무료·내부)
- [~] 3-way 불일치·애매 항목만 = Claude API 2차 재검증 — **교차검증(crossVerify) 구현; 기본은 사내 재검토(보안), Claude 2차는 AGENT_ALLOW_EXTERNAL 시 선택**
- [x] 비용·외부 전송 최소화하면서 정확도 이득은 핵심 지점에 집중
